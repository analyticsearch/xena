About Xena

Xena is designed to assist in the long-term preservation of electronic records. It is capable of converting records from their original data format into open, fully documented formats designed for archival use, then presenting these converted records to users at a later date. It does this through a process we call normalisation. Xena stands for XML Electronic Normalising of Archives.

The long term preservation of electronic records

The long-term preservation of electronic records poses several well known problems. One of these problems is software obsolescence. Over time, older data formats are replaced with newer data formats, and there is no guarantee that we will be able to read files created in these older formats. Newer versions of software replace older versions, creating changes to data formats, even if the name of the format remains the same. Similarly, a software company can go out of business, leaving the proprietary data formats unsupported and thus unreadable within a very short space of time.

Xena is designed to overcome the problem of software obsolesence by converting (or normalising) documents into a vendor neutral open source format based on XML. Normalisation is described in further detail in How Xena preserves records.

As well as converting electronic records into a preservation format, Xena can be used to present normalised electronic records. We use the term presenting rather than viewing records because not all records can be viewed. For instance, we normally listen to a sound file. For further information on presentation, see How Xena preserves records.
