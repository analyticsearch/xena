The Batch Processing Setup Window

Batch normalisation is controlled using the Batch Processing Setup window. This topic explains the elements of the window. How to use this window to conduct batch processing is explained in the other batch processing help topics. To open the Batch Processing Setup window, from the File menu, select Batch Process.

As you can see, the Batch Processing Setup window is made up of several distinct elements. These are:

  • The Menus;
  • The Source, Destination and Normalisation Log panes;
  • The Normalisation Setup Buttons;
  • The Source Directory box;
  • The Destination Directory Setup Controls;
  • The Regular Expression box;
  • The Normalisation Control buttons; and
  • The Progress and Status bars;

These are explained in detail below.

The Menus

The Batch processing setup window has two menus, File and Tools. The options they present are explained below. How to use these options is explained in Using the Batch Processor.

The File menu

The File menu offers three options, Save Log, Save Config and Load Config.

Save Log is used to save the output log of a batch normalisation session as a text file. This lets you examine the log in detail in the text editor of your choice.

Save Config saves the current batch process configuration - ie the groups and their configuration - so that you may use them later. It does not save the list of files to be normalised. This is useful if you will be performing essentially the same or a similar batch process run several times.

Load Config loads a previously saved batch process configuration.

The Tools menu

The Tools menu offers four options, Copy Regexp, Mark Regexp, Copy To Do and Mark To Do.

Copy Regexp is used with regular expressions to select files by name for normalisation. It copies all the files whose name matches the regular expression in the Regular Expression box to the currently selected group in the Destination pane. If there is no currently selected group in the Destination pane then it creates a new group. You can then normalise those files.

Mark Regexp is used with regular expressions to select files by name for normalisation. It highlights all the files whose name matches the regular expression in the Regular Expression box. Unlike Copy Regexp it does not copy them to the Destination pane. This can be useful is you want to select only certain of the files for normalisation.

Copy To Do copies all the files which have not yet been normalised to the currently selected group.

Mark To Do highlights all the files which have not yet been normalised. Unlike Copy To Do it does not copy them to the Destination pane. This can be useful is you want to select only certain of the files for normalisation.

The Source, Destination and Normalisation Log panes

The Source pane shows the directory tree of files and folders that are to be normalised - the source directory. Items are selected selected from this directory for normalisation. If you wish to change the source directory you can do so using the Source Directory box. You can also set it when setting the Xena base directories.

The Destination pane shows the groups of records to be normalised. When you select records from the source pane they are copied to the Destination pane in groups. By creating several different groups you can choose a different normaliser for each group. If you select a group in the Destination pane before you copy items from the Source pane then those items will be copied into the selected group, otherwise Xena will create a new group for them.

The Normalisation Log pane displays the results of the current batch normalisation job. This log shows each file that Xena attempted to normalise and whether that attempt was successful or not. If the attempt was not successful then it will show an error message. The Normalisation Log also includes the start and finish times of the normalisation run, how long it took, how many files were successfully normalised and how many errors were encountered.

The Normalisation Setup Buttons

The Normalisation Setup buttons are used to select files for normalisation from the Source pane and put them into the Destination pane.

The >> button copies the selected items in the Source pane to the Destination pane. If a group has already been selected in the Destination pane then it will copy the items into that group, otherwise it will create a new group.

The << button removes the selected files in the Destination pane and selects them in the Source pane. This can be very slow for large numbers of files. The Clear button in the Destination Directory Setup controls is a better choice when dealing with a large number of files as it simply removes them from the Destination pane.

The Guess button searches for all the files of a particular type using the Xena guesser and copies them to the selected group in the Destination pane, or creates a new group if none are selected. When you click this button Xena will offer you a list of file types. You select one of them and Xena will then go through the files listed in the Source pane and select those matching that file type. You can then select that group and choose the appropriate normaliser for that file type.

The Guess All button searches through all the files in the Source pane using the Xena guesser and copies them to the Destination pane. It will create new groups based on the different file types, one new group for each file type. You can then select each group in turn and choose the appropriate normaliser for that file type.

The To Do button copies all the files in the Source pane that have not yet been normalised into the selected group in the Destination pane, or creates a new group if none are selected. It determines which files have not been normalised by running the file namer on every file in the Source Directory to see if the corresponding file exists in the Destination Directory or not.

The Source Directory box

The Source Directory box is used to select the Source Directory which is displayed in the Source pane. If this is not the directory to be normalised then the Browse button to select the directory to normalise using the following procedure.

  1. Click on the Browse button.
  2. Using the Open window, navigate to and select the source directory.
  3. Click on the Open button.

The correct source directory is now displayed. You can also set it when setting the Xena base directories.

The Destination Directory Setup Controls

The Destination Directory Setup Controls are used to select the Destination Directory which is displayed in the Destination pane. They are also used to configure the normalisers.

If the Destination directory listed is not the desired destination directory then use the Browse button to select the desired destination directory using the following procedure:

  1. Click on the Browse button.
  2. Using the Open window, navigate to and select the destination directory.
  3. Click on the Open button.

The correct destination directory is now displayed. You can also set it when setting the Xena base directories.

The Configure button allows you to set parameters when normalising batches of records. This configuration is done at the level of groups of records, as selected in the output pane of the batch normalisation window. It is normally used as part of Batch normalising by file name. To configure the batch normalisation process for a group of records, select a group and click on the Configure button to open the Batch Configuration window. How to use the Batch Configuration window is described in Using the batch processor.

The Regular Expression box

The Regular Expression box is used to enter regular expressions as part of normalising by file name. For more information, see Batch normalising by file name.

The Normalisation Control buttons

The Normalisation Control buttons are used to run the batch normalising processes. Once you have set up a batch normalisation process you press the Start button to commence it. You press the Stop button to halt the process. The Continue button will restart a halted process, it will only normalise files that have not already been normalised. Next to the Stop button is an icon that swirls while a batch process is running. The Close button is used to close the Batch Procesing Setup window once you have finished batch processing.

The Progress and Status bars

The Progress and Status bars show how a normalisation run is going. The Progress bar shows the progress of the normalisation of each file. The Status bar gives information on the normalisation. It shows whether the normalisation has worked, the time left to conduct the batch run and the name of the file currently being normalised.
