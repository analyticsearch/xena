Normalise Binary File

Xena may be unable to normalise a particular data format because the plugin for that format does not exist. You may also wish to preserve the original bit stream of a data object in addition to normalising it, usually for authentication purposes. We call this XML wrapping, though it is more technically correct to refer to it as normalising a binary file. All computer files are binary data, so any file type can be normalised as a binary file.

XML wrapping involves converting the data object into a Base64 version of the bit stream, then wrapping it in XML containing metadata about the data object. This is a reversible conversion, which means that it is possible to reproduce the original bit stream of the data object from the XML wrapped version, which is not possible with a normalised version. The main purpose of this procedure is to enable a document to be normalised again if there is any question over the normalised version's authenticity.

To normalise a binary file

  1. Open Xena.
  2. From the File menu, select Normalise File.
  3. Using the Open window, navigate to and select the data object that you wish to XML wrap.
  4. Click on Open.
    The Choose File Type dialog box will open, asking you to choose the file type of the document you are opening.
  5. Select Binary from the drop down list and click OK.
  6. Xena will display the Binary Data window.

The data object has now been XML wrapped.

There is no package view for an XML wrapped data object using Xena, the Binary Data window is merely a placeholder to advise you that the window is open.

It is possible to view the document as Raw XML or an XML Tree. See Change View under Xena Menus - The Window Menu for details on how to do this. The object will have to be exported from its XML wrapper and viewed using an appropriate viewing or authoring tool. For how to do this see How to export files under Xena Menus - The File Menu.
