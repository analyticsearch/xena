NAA DATASOURCE DOCUMENT FORMAT

Abstract

This specification documents metadata elements contained within the NAA AIP package wrapper that provide information about the source data object for a preserved digital record.

The elements described in this schema are designed to be enclosed in the meta element of a NAA package instance. They describe attributes of the source data object for the content element of the NAA package instance.

Status

Not released for external comment. Metadata elements implemented in Xena 1.1.17 and later versions.

1. Introduction

This document describes the three metadata elements that form part of the preservation metadata for the NAA AIP for digital records. It is specifically designed to be embedded within a National Archives of Australia package instance to provide extra metadata relating to the original data object the AIP has been created from.

A Datasource should have the namespace declaration: http://preservation.naa.gov.au/datasource/1.0

5. Explanation of elements and attributes

5.1 source-id element

This element contains the dc:identifier (xmlns:dc=http://purl.org/dc/elements/1.1) of the XML wrapped binary AIP that the Xena package is derived from. It is only found in the normalised AIP, where it MUST occur.

The dc:identifier is based on a combination of timestamp and checksum of the location of the original source data object.

XML Code source-id

XML Code <source-id>604e1ed942645e08</source-id>

5.2 Last-Modified element

This element contains the date that the source data object that the Xena package is derived from was last modified.

XML Code last-modified

XML Code <last-modified>2003-05-16T00:09:52</last-modified>

5.1 Type element

This element contains the Mime-type and Sub-type of the source data object that the Xena package is derived from. It is only found in the XML wrapped binary AIP, where it MUST occur.

XML Code type

XML Code <type>image/png</type>
