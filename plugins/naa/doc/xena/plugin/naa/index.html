About the naa plugin

The naa plugin provides Xena with functionality specific to the National Archives of Australia's digital preservation and archival policies.

NAA package wrapper

The naa plugin contains a component for wrapping XML data objects with the NAA XML metadata wrapper. See Setting the Xena properties in the Configuring Xena section of the help for information on how to activate this wrapper. To view the XML schema itself see the Package Document Format schema.

NAA package wrapper metadata elements

The NAA package wrapper contains the following metadata elements:

  • dc:identifier - an identifier for the Xena package instance consisting of a combination of timestamp and checksum of the location of the original source;
  • naa:source-id - normalised file only - the dc:identifier of the XML wrapped AIP which this Xena package is related to;
  • naa:type - the mime type/sub-type of the source data object. Only used for XML wrapped binary data objects;
  • naa:last-modified - the date that the source data object was last modified;
  • dc:source - the pathname of the source data object used to create this Xena package;
  • dcterms:created - the date this Xena package was created; and
  • checksum:checksum - this element, which sits outside the rest of the package, is the MD5 checksum of all the other elements of the package;

Most of these elements, namely naa:source-id, naa:type, naa:last-modified and dc:source, all reside within the naa:datasource element. There is one naa:datasource element for each source data object that was used to create the Xena package. The naa:datasource element for a package is contained within the naa:datasources element.

NAA package view

The naa plugin provides a view for the NAA package wrap format. This is used when you are viewing a normalised file as a Xena package. The Package View is presented as the outermost default view for Xena files when you have activated the NAA Package Wrapper.

NAA file namer

This component implements the output file naming scheme used by the NAA. The NAA naming schema, which is the dc:identifier element of the package wrapper, is designed to prevent duplicate file names. See Setting the Xena properties in the Configuring Xena section of the help for information on how to activate this wrapper.
